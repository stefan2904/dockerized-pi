The `model-test.sh` script is not correctly identifying all error types. 
Currently it checks for "Codex error" and "usage_limit_reached" and standard timeouts, but there are other errors (like the GitHub Copilot 400 "The requested model is not supported") that are being marked as "✅ OK".

Need to:
1. Parse the output properly for standard API error patterns (e.g. 400 Bad Request, 403, 404, 500, etc.)
2. Distinguish between actual successful model output and error strings returned as text.
3. Clean up terminal escape sequences earlier so matching is more reliable.
4. Investigate why `pi -p` hangs on some API failures instead of exiting immediately.

## Additional Observations

- **Codex Quota Exhausted**: Currently, the codex quota is exhausted but the script reports `✅ OK*` with just terminal control sequences instead of properly catching the limit error.
  ```
  | openai-codex/gpt-5.5 | ✅ OK* | ]9;4;3]777;notify;Ready for input;]9;4;0]133;D;0]9;4;3]777;notify;Ready for input;]9;4;0]133;D;0...
  | openai-codex/gpt-5.4 | ✅ OK* | ...
  | openai-codex/gpt-5.3-codex | ✅ OK* | ...
  ```
- **Unsupported Models**: `github-copilot/gpt-5.5` is not enabled in the user's copilot subscription, but it reports `✅ OK` despite outputting an error string.
  ```
  | github-copilot/gpt-5.5 | ✅ OK | ]9;4;3]777;notify;Ready for input;]9;4;0]133;D;0400 The requested model is not supported. |
  ```
- **Hallucinations / Weird Outputs**: Some models are reporting the name of a different model entirely when asked for their name and version.
